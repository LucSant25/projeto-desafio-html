Entendendo comunicação Client x Server

(Voltar)

Nesse módulo aprendemos que de modo geral que o cliente(client) é um dispositivo ou programa que solicita serviços de outro dispositivo, chamado de servidor(Server). O servidor fornece os serviços solicitados pelo cliente.

Aqui está uma visão geral simplificada do fluxo de comunicação entre cliente e servidor:

Requisição (Request) do Cliente:

  • O usuário digita um endereço web (URL) no navegador ou clica em um link.
  • O navegador envia uma solicitação HTTP para o servidor associado ao URL.

Processamento da Requisição no Servidor:

  • O servidor recebe a solicitação HTTP.
  • O servidor processa a solicitação e toma as medidas necessárias para atender à solicitação do cliente.

Resposta (Response) do Servidor:

  • O servidor envia uma resposta HTTP de volta para o navegador do cliente.
  • A resposta pode incluir os dados solicitados, bem como informações sobre o status da solicitação (por exemplo, sucesso, redirecionamento ou erro).

Renderização no Navegador do Cliente:

  • O navegador do cliente recebe a resposta do servidor.
  • Ele interpreta os dados recebidos (como HTML, CSS, JavaScript) e renderiza a página web.

Esse processo é conhecido como o modelo de solicitação-resposta e é a base da comunicação cliente-servidor na web. Além disso, é importante mencionar que, atualmente, muitas aplicações web utilizam tecnologias assíncronas, como AJAX (Asynchronous JavaScript and XML), que permitem a comunicação entre cliente e servidor sem recarregar a página inteira.

Além do HTTP/HTTPS, WebSocket é outro protocolo que permite comunicação bidirecional em tempo real entre o cliente e o servidor, proporcionando uma alternativa mais eficiente para determinados casos de uso, como aplicações em tempo real e jogos online.

Surgimento do HTML

(Voltar)

HTML, que significa Linguagem de Marcação de Hipertexto em inglês (Hypertext Markup Language), teve suas origens no início da década de 1990. Foi desenvolvido por Berners-Lee, um cientista da computação britânico.

Tim Berners-Lee criou o HTML como parte de um projeto mais amplo chamado World Wide Web (WWW). Ele concebeu a World Wide Web como uma maneira de facilitar o compartilhamento de informações entre cientistas em diferentes instituições de pesquisa ao redor do mundo. A ideia era criar um sistema que permitisse a interconexão de documentos através de links hipertexto.

O primeiro documento HTML foi escrito por Tim Berners-Lee em 1991. Ele incluía as tags básicas de marcação, como e . Em 1993, foi lançada a primeira versão pública do HTML, conhecida como HTML 1.0.

À medida que a Web crescia em popularidade, novas versões do HTML foram lançadas, introduzindo melhorias e recursos adicionais. A evolução do HTML continuou com o HTML 2.0, HTML 3.2, HTML 4.01 e XHTML. A versão mais recente é o HTML5, que foi finalizado em 2014 e trouxe consigo uma variedade de recursos avançados, como suporte a multimídia integrado e melhorias na semântica do documento. O HTML5 é amplamente utilizado na criação de páginas web modernas e é suportado pelos principais navegadores.

O que são TAGS

(Voltar)

Em HTML, as "tags" são elementos fundamentais que você usa para estruturar o conteúdo de uma página web. As tags são códigos inseridos no texto para indicar como o navegador deve interpretar e exibir diferentes partes do conteúdo. Elas são envolvidas por colchetes angulares < > e podem ser divididas em tags de abertura e tags de fechamento.

Tag de Abertura:É a tag que indica o início de um elemento. Ela é colocada antes do conteúdo que está sendo marcado.

Tag de Fechamento: É a tag que indica o final de um elemento. Ela é semelhante à tag de abertura, mas inclui uma barra antes do nome da tag.
